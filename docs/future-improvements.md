# Future Improvements After MVP Review

This document lists improvements that are outside the current MVP scope but are recommended for future development.

## Why these are future improvements, not current MVP features

The current SavWise AI MVP focuses on proving the concept for academic review while avoiding risky unfinished production claims. The suggestions below require more time, security review, user testing, and in some cases third-party provider approval before they can be responsibly implemented.

## Suggested Improvements

1. Proper authentication and passwordless login.
2. User roles: normal user, admin, reviewer.
3. Real encrypted session management.
4. Mobile-money API integration only after provider approval.
5. PostgreSQL row-level user isolation or stronger backend access control.
6. Audit logging for privacy-sensitive actions.
7. More advanced transaction categorisation.
8. Better Krio language support reviewed by native speakers.
9. Offline-first mode for low-connectivity users.
10. USSD/SMS support for feature-phone users.
11. Formal privacy impact assessment.
12. Threat modelling and penetration testing.
13. User testing with students and traders.
14. Accessibility audit.
15. Human-reviewed financial literacy content.
16. AI safety evaluation for Wizz responses.
17. Admin dashboard for anonymised aggregate insights.
18. Backup and restore system.
19. Deployment with HTTPS.
20. Open-source community contribution process.
